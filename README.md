# momsitter-2025-4q-web-assignment

맘편한세상 프론트엔드 개발 직무에 지원해주셔서 감사합니다. 본 과제를 통해 향후 맘편한세상의 동료로서 서비스를 더 잘 이해하고, 맘편한세상 서비스 개발 업무를 미리 생각해볼 수 있는 좋은 기회가 되기를 희망합니다.

## Description

Github 사용자 검색 및 즐겨찾기 관리 기능을 갖춘 모바일 웹 애플리케이션을 구현해주세요.

### 기능 명세

- Github 사용자 검색 및 표시
- 즐겨찾기한 사용자를 브라우저 로컬 저장소에 저장
- 즐겨찾기한 사용자를 브라우저 로컬 저장소에서 검색 및 표시
- Github 사용자 검색 화면
  - 검색어 입력창에 검색어를 입력하고, 검색 버튼을 클릭하면 [Github 사용자 검색 API](https://developer.github.com/v3/search/#search-users)를 호출하여 사용자를 검색합니다. 검색 API에 사용되는 필드는 사용자 이름으로 제한합니다.
  - 검색 결과는 최대 100개까지만 보여줍니다. 검색 API 호출 시 page는 1, per_page는 100으로 설정합니다. 자세한 내용은 Github 검색 API 문서를 참고해주세요.
  - Github 사용자 검색 API 응답을 받아 사용자 리스트를 보여줍니다. 각 사용자 별로 프로필 이미지, 사용자 이름, 즐겨찾기 여부를 보여줍니다. 사용자가 이미 즐겨찾기에 추가되어 있으면 이를 표시합니다.
  - 사용자 이름 순으로 정렬합니다. 정렬 순서는 영어, 기타 유니코드 순입니다. 사용자 이름 첫 글자를 헤더를 붙여줍니다.
  - 각 사용자의 아이템 뷰를 누르면 즐겨찾기로 추가합니다. 이미 즐겨찾기된 사용자의 아이템 뷰를 누르면 즐겨찾기를 취소합니다. 본 화면에서 해당 사용자를 즐겨찾기 추가/취소하면, 즐겨찾기 검색 화면에서도 해당 사용자를 추가/삭제합니다.
- 즐겨찾기 검색 화면
  - 검색어 입력창에 검색어를 입력하고 검색 버튼을 클릭하면, 로컬 저장소에서 이름에 검색어가 포함된 사용자를 검색합니다. 검색 필드는 사용자 이름으로 제한합니다.
  - 검색어와 매칭된 모든 사용자 리스트를 보여줍니다. 각 사용자 별로 프로필 이미지, 사용자 이름, 즐겨찾기 여부를 표시합니다. (Github 사용자 검색화면과 동일)
  - 사용자 이름 순으로 정렬합니다. 정렬 순서는 영어, 기타 유니코드 순입니다. 사용자 이름 첫 글자를 헤더를 붙여줍니다.
  - 각 사용자의 아이템 뷰를 누르면 즐겨찾기를 취소합니다. 본 화면에서는 해당 사용자를 삭제하고 Github 사용자 검색 화면에서도 해당 사용자의 아이템 뷰를 갱신합니다.

### 개발 명세

위의 기능 구현을 위한 명세 외에, 아래는 개발면에서 구현해야 할 명세입니다.

- 웹 앱 구동 스크립트 구현
  - package.json 에 start 스크립트를 추가. npm run start 혹은 yarn start 를 통해 웹 앱을 구동할 수 있어야 합니다.
  - 웹 앱 구동시 webpack-dev-server 가 구동이 되고, HMR(Hot Module Replacement)이 적용이 되어야 합니다.
- 웹 앱 빌드 스크립트 구현
  - package.json 에 build 스크립트를 추가. npm run build 혹은 yarn build 를 통해 웹 앱을 빌드할 수 있어야 합니다.
  - 출력 폴더 (e.g. dist/)에는 번들링된 js, html, css 등의 파일이 복사됩니다.
- 웹 앱 미리보기 스크립트 구현
  - package.json에 preview 스크립트를 추가. npm run preview 혹은 yarn preview를 통해 빌드 결과물(dist/ 폴더)을 로컬 서버에서 구동하여 미리 볼 수 있어야 합니다.
  - 정적 파일 서버 라이브러리(eg. serve)는 자유롭게 선택하여 사용합니다.

### 제약 사항

아래 제약 사항은 지원자분의 역량을 검증하고 코드에 대해 깊이 고민해 보기 위한 부분입니다.

- 과제는 바닐라 JavaScript 혹은 TypeScript를 사용합니다.
- React, Angular, jQuery 와 같은 프레임워크나 UI 라이브러리를 사용하실 수 없습니다.
- 프로젝트 생성 및 초기화에 직접적인 도움을 주는 스크립트나 솔루션을 사용할 수 없습니다. (e.g. create-react-app)
- webpack 을 이용하여 소스 번들링이 가능해야 합니다. 다만 환경구축이나 설정 부분을 직접 적용해야 합니다. (e.g. create-react-app 을 통해 자동으로 설정하면
  안됩니다.)
- SPA(Single page application)로 개발되어야 합니다.
- API 호출을 위한 라이브러리는 사용이 가능합니다. (e.g. axios)

### 유의 사항

- 데스크탑 환경은 별도로 고려하실 필요가 없습니다.
- 사용해보지 않은 최신 기술을 익히는 데 시간을 쏟기보다는, 본인에게 가장 효율적인 방법으로 과제를 수행해주세요.
- 주요 클래스 및 함수에 대해 해당 객체의 목적과 기능을 설명하는 1~2줄의 주석을 간략하게 추가해 주세요.
- 과제 진행 과정에 대한 설명을 별도의 README.md에 작성해 주시면 감사하겠습니다. 문제를 어떻게 접근했는지, 어떤 아키텍처를 선택했는지, 어떤 리서치를 했고 어떤 배움을 얻었는지 등 과정 또한 매우 중요하게 생각합니다.
- 과제의 요구사항은 별도의 확인 절차 없이 본인의 판단에 따라 자유롭게 해석하여 진행해 주시면 됩니다. (필요시 관련 내용을 README.md에 기록해 주시면 검토시 적극 참고하겠습니다.)
- AI 도구(ChatGPT, Copilot 등)를 활용하셨다면 어느 단계에서 어떤 목적으로 사용했는지, AI의 결과물을 어떻게 검토하고 반영했는지에 대한 내용을 포함해 주세요. (활용 여부 자체가 감점 요인이 되지는 않으나, 본인의 고민과 문제 해결 과정을 확인하기 위함입니다.)

### 도전 과제 (선택사항)

- API 호출 시 발생하는 HTTP 에러를 처리하고 사용자에게 피드백을 제공합니다.
- 사용자 클릭 시 모달 팝업을 띄워 해당 사용자의 GitHub Repository 목록을 표시합니다.

## Criteria

지원자분이 구현한 코드를 바탕으로 다음 요소들을 중점적으로 검토하게 됩니다.

- 서비스 기획 내용에 대한 정확한 이해
- 기능 요구사항에 대한 효율적이고 적시적인 구현
- 버그를 최소화하는 높은 코드 품질
- 유연한 변경 사항 대응을 위한 재사용 및 확장 가능한 애플리케이션 구조 설계
- 협업을 위한 코드 및 문서 가독성
